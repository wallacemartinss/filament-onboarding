# Screenshots

The main README references these files by name — drop each image here and uncomment
its slot in the README (search for `docs/images/` there; every slot is an HTML comment).

Suggestions that keep the set looking like one set: same panel, same theme (light reads
better on GitHub), a window around 1440–1600 px wide, and real-looking content — a journey
half-done tells the story better than one at 0%.

| File | What to capture |
|------|-----------------|
| `principal.png` | The hero. The floating checklist open over a real dashboard, journey half-done — the one image that explains the package. |
| `welcome.png` | The welcome screen over the first page after login: title, step count, the three answers. |
| `launcher.png` | The floating checklist open: some steps ticked, one highlighted as next, a step with a video button. Tabs visible if you have two journeys. |
| `tour.png` | A tour mid-flight: spotlight on a field, popover with counter and buttons — ideally inside a wizard. |
| `media-modal.png` | The video modal docked in a corner (bottom-right), playing, with the watched-percentage footer, page usable behind it. |
| `progress-page.png` | The progress page: ring with percentage, done/left/skipped counters, the next-step call to action, step tiles below. |
| `authoring-step.png` | The step editor in the admin panel — the tour tab with a couple of stops, or the media tab with a video configured. |
