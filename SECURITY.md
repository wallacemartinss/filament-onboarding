# Security Policy

## Supported Versions

Security fixes are applied to the latest `2.x` release. Please make sure you are
running the most recent version before reporting an issue.

| Version | Supported          |
| ------- | ------------------ |
| 2.x     | :white_check_mark: |
| < 2.0   | :x:                |

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues,
discussions, or pull requests.**

Instead, report them privately through one of these channels:

- **GitHub Security Advisories** (preferred) — open a private report at
  [Security → Report a vulnerability](https://github.com/wallacemartinss/filament-onboarding/security/advisories/new).
- **Email** — [wallacemartinss@gmail.com](mailto:wallacemartinss@gmail.com).

Please include as much detail as you can:

- the affected version(s),
- a description of the vulnerability and its impact,
- steps to reproduce or a proof of concept,
- and any suggested remediation, if you have one.

## What to Expect

- **Acknowledgement** within 72 hours of your report.
- An initial assessment and, if confirmed, a plan and timeline for a fix.
- Progress updates as the fix is developed.
- Public disclosure and credit (if you wish) once a patched release is available.

Thank you for helping keep this project and its users safe.
